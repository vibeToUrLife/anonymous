# 🏇 全站赛马（万能决策器）— 设计文档

日期：2026-07-21
状态：已确认（纯比赛版；全站一场广播式；全部是手绘马）

## 1. 目标

一个"赌马式"的决策小游戏：任何人写下一堆选项（比如今天喝什么、谁请客），每个选项由一匹马带着跑，速度随机，跑完显示完整排名。全站所有在线的人**实时看到同一场比赛的同一画面**。入口和界面都在 `index.html`（主留言板）。

## 2. 范围

**v1 包含**：发起比赛（2~12 个选项）→ 15 秒倒计时全站广播 → 约 22~32 秒赛马动画 → 完整排名榜。手绘卡通马。全触屏操作（mobile-first）。

**v1 不包含**（结构上预留）：金币下注（未来挂在 `current/bets/{uid}`，倒计时窗口就是下注窗口）；防内定的种子机制（见 §9）。

## 3. 数据模型（RTDB）

只有一个节点，天然保证"全站同时只有一场"：

```
horse_race/current = {
  id:      "r1721551234",          // 本场唯一 id（时间戳字符串）
  options: ["奶茶","咖啡","柠檬茶"], // 2~12 个，每个 1~24 字
  seed:    8371023,                 // 整数随机种子 = 全场比赛的"剧本"
  startAt: 1721551249000,           // 开跑时刻（校准后的服务器毫秒时间 + 15s）
  by:      "<uid>",
  byName:  "发起人昵称"             // ≤40 字
}
```

- 新比赛直接覆盖旧记录；比赛结束后记录留着，后来的人打开能看到"上一场结果"。
- 不需要 history 节点（v1 不做历史）。

## 4. 实时同步机制（核心）

**种子确定性回放**：开赛只写一条上面的小记录。所有客户端 `onValue` 监听 `horse_race/current`，各自用同一个 `seed` 跑同一套确定性模拟。

- 马在 t 时刻的位置是纯函数 `pos(seed, 马号, t)`，与帧率无关（30fps 手机和 144fps 电脑看到的位置一致）。
- 时间基准：`db.ref('.info/serverTimeOffset')` 校准出服务器时间，倒计时与赛程都由 `serverNow() - startAt` 驱动 → 各端画面帧帧一致。
- 一场比赛全程只有 1 次网络写入；发起人关页面不影响比赛。
- 比赛中途打开页面的人从正确进度接着看；结束后打开的人直接看到排名。

## 5. 比赛模拟算法

用 mulberry32(seed ⊕ 马号×大质数) 为每匹马生成：

1. **冲线时间** `F_i`：均匀分布在 22~32 秒 → 排名 = `F_i` 升序（开赛瞬间已注定，动画只是演出来）；同值按马号破平（确定性）。
2. **位置曲线**：`pos(t) = t/F_i + wobble(t)`，wobble 为 2~3 个随机相位/频率的正弦和，端点归零、幅度受控保证**单调只进不退** → 自然出现领先、被反超、末段冲刺逆转。
3. 马的**奔跑步频**与当前瞬时速度成正比（冲刺时腿摆得更快）。

常量：倒计时 15s；冲线 22~32s；赛道 2~12 道。

## 6. 马的绘制（手绘，不用 emoji）

跟站内宠物（`games/pets/*.js`）同一家族的 canvas 卡通风格，侧面奔跑马：

- **形体**：椭圆身体 + 前伸的脖子 + 头（带口鼻）+ 双耳 + 眼睛；深色鬃毛沿颈后飘；尾巴后飘并随奔跑摆动。
- **四条腿**做奔跑循环（前后腿对摆 + 收腿相位），身体带 `|sin|` 上下颠簸；身后飘小灰尘团。
- **配色**：12 套马毛色板（栗、黑、白、金、灰、枣红、花斑等），按道次固定分配；身上搭**鞍布写道次号**。
- 选项文字跟随在马上方（超长省略号）；冲线后马旁出现名次徽章（🥇🥈🥉/数字）。
- 整条赛道一块 canvas（DPR 缩放），比赛期间 rAF 逐帧重绘；固定全赛道视角（不跟随镜头），右侧黑白格终点线。营火（campfire.js）已验证这种整画布重绘模式。

## 7. UI 流程（全触屏）

- **入口**：「✨ 更多玩法」加 tile：🏇 赛马 · 万能决策（id `horseRaceBtn`）。
- **overlay**（金币乐园同款全屏模式）四个状态：
  1. **发起页**：大输入框"一行一个选项"（2~12 行，超出提示不截断）、大按钮「🏁 开赛」；有比赛进行中时按钮置灰显示"比赛进行中"。若 `current` 是已结束的比赛，下方显示"上一场结果"。
  2. **倒计时**："XX 发起了比赛" + 选项列表 + 15 秒大倒计时。
  3. **比赛中**：横向多道赛道，马从左往右跑。
  4. **结果**：完整排名榜（名次 + 选项 + 用时），停留到用户关闭；任何人可发起下一场。
- **广播**：登录用户页面上出现新比赛（新 `id` 且未结束）→ 自动弹出 overlay；右上角 ✕ 关闭，`sessionStorage` 记住已忽略的 raceId，同一场不再弹；新一场照弹。用户也可随时从 tile 手动进入。
- 手机竖屏：道高压缩自适应，12 道也放得下。

## 8. 安全规则（`database.rules.json` 新增，需贴到 Firebase Console）

```json
"horse_race": {
  "current": {
    ".read": "auth != null",
    ".write": "auth != null && (!data.exists() || now > data.child('startAt').val() + 60000) && newData.hasChildren(['id','options','seed','startAt','by'])",
    "id":      { ".validate": "newData.isString() && newData.val().length <= 30" },
    "seed":    { ".validate": "newData.isNumber()" },
    "startAt": { ".validate": "newData.isNumber() && newData.val() >= now - 5000 && newData.val() <= now + 30000" },
    "by":      { ".validate": "newData.val() == auth.uid" },
    "byName":  { ".validate": "newData.isString() && newData.val().length <= 40" },
    "options": {
      ".validate": "newData.hasChildren(['0','1']) && !newData.child('12').exists()",
      "$i": { ".validate": "newData.isString() && newData.val().length >= 1 && newData.val().length <= 24" }
    },
    "$other": { ".validate": false }
  }
}
```

要点：登录才能读写；**比赛进行中禁止开新场**（只有开跑 60 秒后才可覆盖，最长赛程 32s < 60s）；`startAt` 必须落在 `now±` 窗口；`by` 必须是本人。

## 9. 已知取舍

- seed 由发起人客户端生成，懂技术的人理论上可挑 seed 内定冠军。纯娱乐版接受；将来接金币下注时改为承诺-揭示（commit-reveal）或云函数发种子，数据结构已兼容。
- 覆盖 `current` 意味着上一场结果被冲掉——v1 接受（想看历史属于未来扩展）。

## 11. 测试计划

1. **确定性**：同一 seed 在两个浏览器窗口跑出逐帧一致的画面与相同排名（本地双窗对照）。
2. **中途加入**：比赛进行中刷新页面 → 从正确进度接看；结束后打开 → 直接见排名。
3. **规则**：贴规则前本地验证写入 shape；贴 Console 后验证"进行中不能开新场"、未登录不可读写。
4. **手机**：竖屏 12 道可读可点（老流程：本地测好 → 贴规则 → push origin/master 即部署）。
